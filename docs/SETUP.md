# 리포트서머리 매일 자동 실행 설정 가이드

매일 아침 9시(KST)에 GitHub Actions가 클라우드에서 크롤링을 실행하고,
결과를 구글 드라이브의 `리포트서머리.xlsx`에 누적 저장합니다.
**내 PC가 꺼져 있어도 실행됩니다.**

아래 순서대로 딱 한 번만 설정하면 됩니다.

---

## 1단계. 구글 서비스 계정 만들기 (드라이브 저장용 열쇠)

GitHub Actions가 내 드라이브에 파일을 쓰려면 "서비스 계정"이라는 로봇 계정이 필요합니다.

1. https://console.cloud.google.com 접속 → 로그인
2. 상단에서 **새 프로젝트** 생성 (이름 예: `report-summary`)
3. 좌측 메뉴 → **API 및 서비스 → 라이브러리** → `Google Drive API` 검색 → **사용 설정**
4. 좌측 메뉴 → **API 및 서비스 → 사용자 인증 정보**
   → **사용자 인증 정보 만들기 → 서비스 계정**
   → 이름 입력(예: `report-bot`) → 완료
5. 만들어진 서비스 계정 클릭 → **키** 탭 → **키 추가 → 새 키 만들기 → JSON**
   → JSON 파일이 다운로드됩니다. (이 파일 내용을 나중에 GitHub에 넣습니다)
6. 서비스 계정의 **이메일 주소**를 복사해 둡니다.
   (형식: `report-bot@report-summary.iam.gserviceaccount.com`)

---

## 2단계. 드라이브 파일을 서비스 계정과 공유

서비스 계정은 기본적으로 내 드라이브에 접근 못 합니다. 파일을 "공유"해줘야 합니다.

1. 구글 드라이브에서 `Colab Notebooks/리포트서머리.xlsx` 파일을 우클릭 → **공유**
2. 1단계에서 복사한 **서비스 계정 이메일**을 추가하고 권한을 **편집자**로 설정 → 완료
3. 그 파일을 열고 주소창 URL에서 **파일 ID**를 복사합니다.
   `https://drive.google.com/file/d/`**`여기가_파일ID`**`/view` 형태입니다.
   (또는 스프레드시트로 열리면 `/d/파일ID/edit`)

> ⚠️ 현재 `리포트서머리.xlsx`는 pandas가 만든 **엑셀(.xlsx) 파일**이어야 합니다.
> 혹시 "구글 스프레드시트"로 변환되어 있으면, 다운로드했다가 .xlsx로 다시 올려주세요.

---

## 3단계. GitHub 저장소 만들고 코드 올리기

1. https://github.com 에서 새 저장소(repository) 생성 (예: `report-summary`, **Private** 권장)
2. 이 폴더(`클로드코드`)를 그 저장소에 올립니다. (아래 4단계 명령 참고)

---

## 4단계. GitHub Secrets 등록 (비밀값 2개)

저장소 페이지 → **Settings → Secrets and variables → Actions → New repository secret**

| 이름 | 값 |
|------|-----|
| `GDRIVE_SA_KEY` | 1단계에서 받은 **JSON 파일 전체 내용**을 그대로 붙여넣기 |
| `GDRIVE_FILE_ID` | 2단계에서 복사한 **파일 ID** |

---

## 5단계. 동작 확인

1. 저장소 → **Actions** 탭 → `Daily Report Summary` 선택
2. **Run workflow** 버튼으로 수동 실행 (스케줄을 기다리지 않고 즉시 테스트)
3. 초록불이 뜨고 로그에 `✅ 구글 드라이브 업데이트 완료`가 보이면 성공
4. 이후에는 **매일 아침 9시경 자동 실행**됩니다.

---

## 참고

- 스케줄 시간을 바꾸려면 `.github/workflows/daily-report.yml`의 `cron` 값을 수정하세요.
  (`0 0 * * *` = UTC 0시 = KST 9시. 예: KST 8시로 바꾸려면 `0 23 * * *`)
- GitHub Actions 무료 사용량(공개 저장소 무제한 / 비공개 월 2,000분) 안에서 충분히 돌아갑니다.
- 월간 차트(노트북 cell-7)는 이 자동화에는 포함하지 않았습니다. 필요하면 추가해드릴 수 있어요.
